Run overview

In plain English: after each training session, the model is graded on a set of examples it has never seen (the test set). Accuracy is the share of those examples it got right — 0.70 means 70 out of 100 correct. F1 blends two ideas: how often the model's predictions are correct, and how many real cases it manages to find. The macro version grades every class equally (a fair test when some classes are rare); the weighted version lets the big classes count for more. Arrows compare this run with the previous one.

Performance by class

In plain English: for each class — Precision: when the model says "this is class X", how often it is right. Recall: of all the real class-X cases, how many it catches. F1: the balance of the two (high only when both are high). Support: how many test examples the class has — scores based on very few examples are less reliable. Classes flagged Weak (F1 < 0.60) deserve attention first.

Class distribution

In plain English: this shows how many examples each class has in the dataset. When one class dominates, the model naturally gets better at it and worse at the rare ones — that is called class imbalance, and it is the usual reason rare classes score low above. The imbalance ratio compares the biggest class with the smallest.
